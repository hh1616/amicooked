# Am I Cooked? — Design Specification

**Version:** 1.0  
**Last updated:** 2026-04-02  
**Status:** Ready for review

---

## 1. Overview

**Am I Cooked?** is an AI Job Risk Calculator at **amicooked.me**. Users enter their job title, get a "cooked score" (0–100) showing how likely AI is to replace them, plus a breakdown of what's safe and what's not. Designed to be fun, shareable, and screenshot-optimized.

**Architecture:** Static single-page site. Curated JSON dataset (~200 jobs). No backend, no cookies, hostable anywhere.

---

## 2. Visual Style

**Brutalist web design** — thick borders, bold typography, high contrast, opinionated layout. Deliberately anti-AI aesthetic: asymmetry, personality in copy, texture, cultural references. Nothing should look like it was generated by AI.

---

## 3. Typography

| Role | Font | Weight | Notes |
|---|---|---|---|
| Display headings | Barlow | 900 (Black) | "AM I COOKED?", "CHECK", verdict text |
| Body / labels | Space Mono | 400, 700 | Monospace. All body text, input fields, meta labels |
| Score number | Instrument Serif | 400 italic | The big score number only (e.g. "34") |

**Google Fonts import:**
```
Space+Mono:wght@400;700
Barlow:wght@800;900
Instrument+Serif:ital@0;1
```

**Sizing (mobile → desktop):**

| Element | Mobile | Desktop |
|---|---|---|
| Score number | 60px | 96px |
| Score /100 | 22px | 32px |
| Verdict | 24px | 36px |
| Job label | 9px | 11px |
| Stat box values | 18px | 26px |
| Stat box labels | 6px | 8px |
| Section headings | 12px | 14px |
| List text | 8px | 11px |
| Quips | 7px | 9px |
| Header "AM I COOKED?" | 11px | 14px |
| Button labels | 10px | 13px |

**Spacing quirks:**
- Space Mono needs `word-spacing: -2px` at small sizes (8px) to look right
- Quips use `word-spacing: -1px`
- Landing page H1 uses `letter-spacing: -1px` and `font-size: clamp(64px, 14vw, 110px)`

- Phone frame presentation: 260px wide, rounded corners, cream bg

- Black header bar: "AM I COOKED?" left, red dot right

**Desktop** (`results-desktop-v1.html`)

## 6. Components

### 6.1 Stat Boxes
- Flex row, 3 equal boxes
- Borders: 2px (mobile) / 3px (desktop), solid black
- Adjacent boxes share borders (border-left: 0 on 2nd and 3rd)
- Value in Barlow 900, label in Space Mono

### 6.2 Numbered List (Saving Graces)
- Marker column: 22px (mobile) / 28px (desktop)
- Numbers in `#999`: 01. 02. 03. 04.
- Select items get quips in parentheses, `#aaa`, smaller font
- Not every item gets a quip — selective humor only

### 6.3 Cross List (What's Getting Cooked)
- Same marker column width
- Red cross `✕` in `#cc0000`, bold (700), slightly larger than text
- Vertically aligned with numbered markers
- `position: relative; top: -0.5px` for alignment nudge

### 6.4 Download/Share Buttons
- Two equal-width buttons in a flex row
- Download: white bg, black border, download tray SVG icon
- Share: black bg, white text, chain link SVG icon
- Share hover: `#ff0000` background
- Icons: 11px (mobile) / 14px (desktop), stroke-width 2.5

### 6.5 Header Bar
- Full width, black background
- "AM I COOKED?" left in Barlow 900
- Red dot right (pulsing on landing, static on results)

### 6.6 Footer Bar
- Landing: fixed bottom, cream bg, 1px top border
- Results mobile: black bg within phone frame
- Results desktop: cream bg, 1px top border (matches landing)

## 7. Data Schema

Each job in the JSON dataset needs:

```json
{
  "id": "marketing-manager",
  "title": "Marketing Manager",
  "score": 34,
  "verdict": "Lightly Toasted",
  "verdictQuip": "some parts of your job are already on borrowed time.",
  "timeline": "~5",
  "timelineUnit": "YRS",
  "automatable": "62%",
  "pivotPotential": "HIGH",
  "savingGraces": [
    { "text": "Creative strategy", "quip": "robots can't vibecheck" },
    { "text": "Brand storytelling", "quip": null },
    { "text": "Client relationships", "quip": "they're needy" },
    { "text": "Cross-functional leadership", "quip": null }
  ],
  "gettingCooked": [
    { "text": "Reporting & analytics", "quip": null },
    { "text": "A/B testing setup", "quip": null },
    { "text": "Basic copywriting", "quip": "first to go, sorry" }
  ]
}
```

### Verdict scale (score → label → one-liner):
| Score Range | Verdict | One-liner |
|---|---|---|
| 0–15 | Raw | you're untouchable. for now. |
| 16–30 | Rare | AI's circling but hasn't landed. |
| 31–45 | Lightly Toasted | some parts of your job are already on borrowed time. |
| 46–60 | Half Baked | it's a coin flip. better have a plan B. |
| 61–75 | Well Done | the robots are warming up your seat. |
| 76–90 | Extra Crispy | most of what you do, a bot does cheaper. |
| 91–100 | Fully Cooked | update your LinkedIn. yesterday. |

### Quip tone rules:
- Quips must punch at the **technology or the situation**, never at the person doing the job
- No implying a job is worthless — even fully cooked jobs have dignity
- No gendered/racial assumptions about who does a job
- When in doubt, drop the quip (set to null)
- Validated automatically by `tools/validate_jobs.py` (punch-down pattern detector)

### Disclaimer:
All results pages include: `*for entertainment — not career advice` in muted text below the action buttons.

## 8. Interactions

- **Job input:** Fuzzy search / autocomplete against the JSON dataset
- **Check button:** Transitions from landing to results view
- **Download:** Uses html-to-image library to generate a PNG of the results card
- **Share:** Copies a share link to clipboard (or native share on mobile)
- **TRY ANOTHER:** Returns to landing page / clears input

---

## 9. Sharing & Screenshot Optimization

The results card is designed to be screenshot-friendly:
- High contrast, readable at small sizes
- Mobile results fit in a single screenshot
- Downloadable PNG via html-to-image captures the results card
- No Apple emojis — SVG icons only

---

## 10. Constraints

- **No Apple emojis** — use SVG icons or text symbols
- **No AI-looking design** — no gradients, no symmetric layouts, no generic sans-serif
- **No round fonts** — they clash with Space Mono's angular character
- **Selective humor** — quips on some items, not all. Less is more.
- **Quips punch at tech, not people** — mock the situation or the technology, never the worker. Enforced by validator.
- **Static site** — single HTML file, no backend, no frameworks
- **No cookies, no tracking** — privacy-first
- **Disclaimer required** — `*for entertainment — not career advice` on every results page
